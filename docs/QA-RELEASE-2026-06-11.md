# Auralith Maps: предрелизный QA-отчет

Дата проверки: 11 июня 2026  
Стенд: https://park.auralith.ru/  
Браузер: Chrome 149, desktop 1440x1000 и mobile 390x844  
Результат: **RELEASE BLOCKED**

## Итог

Релиз не рекомендуется до исправления двух блокирующих проблем:

1. 51 из 65 уникальных фотографий парковок не загружаются (`404`).
2. После потери GPS навигатор продолжает показывать старую скорость и маневр без предупреждения.

Дополнительно требуется проверить задержку перестроения маршрута и офлайн-режим PWA.

## Найденные дефекты

### QA-001. Массово отсутствуют фотографии парковок

Критичность: **High / Release blocker**

- В админке 308 точек, у 38 точек указаны фотографии.
- Проверено 65 уникальных URL фотографий.
- Загружаются 14 файлов.
- Не загружается 51 файл, то есть 78,5%.
- Ошибочные URL отвечают `404`, преимущественно из `/storage/photos/...`.

Ожидается: каждая опубликованная фотография доступна либо заменена корректным placeholder.

Фактически: в данных сохранены ссылки на отсутствующие файлы.

Доказательство: `artifacts/qa/live-report.json`.

### QA-002. При потере GPS остаются старая скорость и маневр

Критичность: **High / Release blocker**

Шаги:

1. Построить маршрут и начать движение.
2. Передать скорость 50 км/ч.
3. Сделать геолокацию недоступной.
4. Подождать 7 секунд.

Фактически:

- Навигация остается активной.
- На экране остается скорость 50 км/ч.
- Продолжает отображаться старый маневр.
- Предупреждение о потере GPS отсутствует.

Обработчик ошибки обновляет `updatedAt` старой позиции и повторно рисует старые данные:  
`resources/js/modules/parking-form.js:2594`.

Ожидается: заметное сообщение «GPS недоступен», признак устаревших данных и сброс/затемнение скорости.

Доказательства: `artifacts/qa/reroute-report.json`, `14-mobile-navigation-gps-unavailable.png`.

### QA-003. Перестроение после отклонения начинается с заметной задержкой

Критичность: **High**

- После сильного отклонения запрос нового маршрута начался через 11,6 секунды.
- Ответ API получен еще через 0,6 секунды.
- Полное наблюдаемое время: около 12,2 секунды.
- Повторные перестроения дополнительно ограничены cooldown 25 секунд:  
  `resources/js/modules/parking-form.js:2664`.

Ожидается: начало перестроения в пределах 2-5 секунд после подтвержденного отклонения.

Доказательства: `artifacts/qa/reroute-report.json`, `15-mobile-navigation-rerouted.png`.

### QA-004. Маневр «0 м» не сменился при движении

Критичность: **Medium**

В GPS-эмуляции скорость корректно менялась 30 → 40 → 50 км/ч, но три последовательных обновления продолжали показывать один маневр с расстоянием `0 м`.

Ожидается: после прохождения точки маневра выбирается следующая инструкция.

Требуется повторить на реальном устройстве и маршруте, проходящем точно по дорожной геометрии.

Доказательство: `artifacts/qa/live-report.json`.

### QA-005. Офлайн-сообщение PWA отображается битой кодировкой

Критичность: **Medium**

Service worker активен, приложение загружается без сети, но:

- текст ошибки отображается как mojibake;
- парковки отсутствуют;
- показывается только оболочка и ранее закэшированная карта.

Ожидается: читаемое сообщение о недоступности сети. Если офлайн-список не поддерживается, это должно быть явно объяснено.

Доказательство: `artifacts/qa/19-mobile-offline.png`.

### QA-006. Часть мобильных элементов меньше 44x44 px

Критичность: **Medium / Accessibility**

Примеры:

- кнопки карты: 36x36 px;
- кнопки карточки: 38x38 px;
- checkbox выбора для экспорта: 22x22 px;
- часть строк списка имеет высоту 35 px.

Это повышает вероятность ошибочных нажатий в автомобиле и не соответствует рекомендуемому размеру touch target.

### QA-007. Не загружается шрифт подписей карты

Критичность: **Low**

Запрос `Open Sans Bold/0-255.pbf` возвращает `404`. MapLibre использует локальный fallback и пишет предупреждения в консоль.

### QA-008. Отсутствуют основные security headers

Критичность: **Medium / Hardening**

На главной странице отсутствуют:

- `Content-Security-Policy`;
- `Strict-Transport-Security`;
- `X-Content-Type-Options`;
- `Referrer-Policy`;
- защита от внешнего framing (`frame-ancestors 'self'` подходит для внутреннего iframe).

Админ-панель без cookie корректно отвечает `302` и перенаправляет на главную.

## Ограничения проверки

- Фактическая установка на физический Android/iPhone не выполнялась. Chrome подтверждает техническую устанавливаемость PWA.
- Компас, реальный шум GPS, фоновой режим ОС и энергосбережение требуют полевого теста на устройствах.
- Удаление точек, массовое изменение статусов и изменение ролей пользователей намеренно не выполнялись на production-данных.
- Загрузка новой фотографии на production не выполнялась, чтобы не создавать тестовые данные.
